# Google Sheets 後台結構說明

## 專案名稱
QR Code 飲料兌換券管理系統

## 1. Google Sheets 總覽

本系統以 Google Sheets 作為核心資料庫，負責儲存會員資訊與其飲料兌換券的剩餘數量。管理者可直接透過 Google Sheets 進行會員資料的新增、修改與查詢，無需額外的後台管理介面。Google Apps Script 將作為中介層，處理前端兌換頁面與 Google Sheets 之間的資料互動。

## 2. 資料表結構：`Members` 工作表

`Members` 工作表是系統中唯一的資料表，用於儲存所有會員的詳細資訊。

## 3. Google Apps Script 程式碼架構

Google Apps Script 將部署為 Web App，提供 API 接口供前端兌換頁面呼叫，實現會員資訊查詢與剩餘杯數更新的功能。

### 3.1 核心功能

*   **`doGet(e)`**: 處理 `GET` 請求，用於查詢會員資訊。
    *   接收 `id` 參數 (Member ID)。
    *   從 `Members` 工作表查詢對應會員的 `Name` 和 `Remaining`。
    *   返回 JSON 格式的會員資訊，或錯誤訊息。

*   **`doPost(e)`**: 處理 `POST` 請求，用於更新會員的剩餘杯數。
    *   接收 JSON 格式的請求主體，包含 `id` 參數 (Member ID)。
    *   從 `Members` 工作表查詢對應會員的 `Remaining`。
    *   檢查 `Remaining` 是否大於 0，若否則返回錯誤。
    *   將 `Remaining` 減 1 並更新回工作表。
    *   返回 JSON 格式的更新結果，包含新的 `Remaining`，或錯誤訊息。

### 3.2 輔助功能

*   **`getMembersSheet()`**: 取得 `Members` 工作表的實例。
*   **`findMemberRow(memberId)`**: 根據 Member ID 查找會員所在的行數。
*   **`getMemberData(row)`**: 取得指定行數的會員資料。
*   **`updateRemaining(row, newRemaining)`**: 更新指定行數的 `Remaining` 欄位。

## 4. 部署與設定

1.  在 Google Sheets 中，點擊「擴充功能」>「Apps Script」開啟腳本編輯器。
2.  將提供的 Apps Script 程式碼貼入編輯器中。
3.  點擊「部署」>「新增部署作業」，選擇「網路應用程式」。
4.  設定「執行身分」為「我」，「誰可以存取」為「任何人」。
5.  部署後，將會獲得一個 Web App URL，此 URL 將用於 QR Code 的生成與前端兌換頁面的呼叫。

## 5. 錯誤處理與 Edge Case

Apps Script 程式碼將會包含以下錯誤處理機制：

*   **查無會員**: 當提供的 Member ID 不存在時，返回特定的錯誤訊息。
*   **剩餘杯數為 0**: 當會員的 `Remaining` 為 0 時，阻止兌換並返回提示訊息。
*   **重複提交**: 透過前端的按鈕禁用與 Loading 狀態，配合後端檢查，防止短時間內重複扣點。
*   **資料異常**: 對於 `Remaining` 欄位的空值、負數或非數字格式，進行檢查與處理。
*   **併發問題**: 雖然 Google Apps Script 對於單一儲存格的寫入有原子性，但仍需考慮多個店員同時操作同一會員時的競態條件，並在程式碼中進行適當的鎖定或重試機制（若有需要）。

## 6. QR Code 內容格式

QR Code 將包含 Web App 的 URL 與 Member ID 參數，例如：

`https://your-webapp-url/exec?id=A001`

其中 `your-webapp-url/exec` 是部署後的 Google Apps Script Web App URL，`id` 參數為會員的 `Member ID`。
